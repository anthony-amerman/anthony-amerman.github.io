JAVASCRIPT FUNDAMENTALS: A non-technical explanation

JavaScript is kind of like the fingers of a website. It file of action that holds together HTML and CSS, and it can also build things on top of them or insert or change things inside them. It takes useful data -- user profile information or store stock, for example -- and stores them as "objects" in a way that is convenient to use. JavaScript also deals with keeping track of actions performed on the page -- clicks, keystrokes, other events -- and deciding what to do when they occur, whether that is performing an action itself, giving an output, or just sitting back and listening. it performs its tasks using what are called "functions".

Control flow is the order in which JavaScript code is run. It basically runs top to bottom unless it is told otherwise. One thing that can change the flow is a loop. we perform loops in our everyday lives. imagine you don't care about collosal wastes of land and water and decide to go golfing. the way you keep score on a score card is much like a loop. you start at the first hole, wack the ball to the end, and record your score. then you go to the second hole and do the same thing. you cycle through these actions for every hole until there are no more holes left, at which point you exit this loop. here is an example of a loop in JavaScript:

When a loop is enountered, it is:
  1. entered and given an initial statement, here, let i = 0, starts a counter at 0
  2. given a pre-defined condition. here, if i < 10, the parts inside curly brackets "{}" re performed
  3. here, console.log(i), prints out which iteration of the loop we are performing
  4. at the end of one cycle the cycle complete instruction is performed. here, i++, adds 1 to the counter [i]
steps 2-4 are repeated until the condition is not met -- here, until [i] is no longer less than 10 -- at which point the loop is broken and continues down through the rest of the code.

The Document Model Object or, DOM, is an interface a web browser or other program uses to structure all of the information in a web document, arranging it into a heirarchical tree, that can be manipulated to change its content and the way in which that content is presented. One way to interact with the DOM is to use developer tools built into most browsrs. press f12 or ctrl + shift + i to open them on this page. we are now "looking under the hood" of a web page. using this we can look at the code of a website or test new things out.

Functions are the main "doers" in JavaScript. They are first defined (built) and then can be called (put into action) later on. they can be used to pull data from an array or object, send it somehwere else, take an input and add or subtract things to it and give you an output, send a message when a button is clicked. pretty much any interactivity on a webpage is built form functions in Javascript. they allow us to manipulate and change the document based on events and inputs.
